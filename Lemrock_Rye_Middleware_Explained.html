1 왜 "미들웨어"가 필요한가?

AI 에이전트 시대의 새로운 문제와, 그것을 푸는 두 가지 방법

🏬 비유로 이해하기: "해외 백화점에 입점하고 싶은 한국 브랜드"

🏪

한국 브랜드 (판매자)

좋은 상품은 있지만, ChatGPT·Gemini·Perplexity 같은 "AI 백화점"에 어떻게 입점하는지 모릅니다. 각 AI 플랫폼마다 형식이 다르고, 언어도 다르고, 결제 시스템도 다릅니다.

→
🔌

미들웨어 = "무역 대행사"

브랜드 대신 각 AI 백화점의 입점 서류를 작성하고, 상품을 진열하고, 결제를 처리하고, 성과를 보고해주는 중간 대행 인프라입니다.

→
🤖

AI 에이전트 (새로운 백화점)

ChatGPT, Claude, Gemini, Perplexity, Copilot... 각각 다른 규칙(ACP, UCP, MCP)으로 상품을 받습니다. 미들웨어가 이 차이를 흡수합니다.

💡 핵심 문제: "프로토콜 파편화"

2026년 5월 현재, AI 커머스 프로토콜이 난립하고 있습니다: OpenAI+Stripe의 ACP, Google+Shopify의 UCP, Visa의 Intelligent Commerce(TAP), Mastercard의 Agent Pay. 각 프로토콜마다 피드 형식, 결제 방식, 온보딩 절차가 전부 다릅니다. 실제로 UCP/ACP 듀얼 구현 가맹점이 단일 프로토콜 대비 40% 더 많은 에이전트 트래픽을 확보한다는 초기 데이터도 나왔습니다. 브랜드가 이걸 하나하나 직접 대응하는 건 비현실적이며, OpenAI조차 ChatGPT 네이티브 체크아웃을 12개 머천트 단계에서 축소하고 앱 기반 결제로 전환할 정도로 프로토콜 온보딩이 어렵습니다. 그래서 "한 번 연동하면 모든 AI에 연결되는 미들웨어"가 더욱 중요해졌습니다.

🔍 Lemrock과 Rye는 이 문제를 다른 지점에서 풉니다

Lemrock = "AI가 우리 상품을 발견하게 하는 미들웨어" → 상품 카탈로그를 AI가 이해하는 형태로 변환·배포·추적 (ACP/UCP 규격 자동 대응)
Rye = "AI가 어디서든 결제를 완료하게 하는 미들웨어" → 어떤 쇼핑몰이든 URL만 있으면 결제를 자동 처리 (+ Skyfire KYA로 에이전트 신원 인증)

2 Lemrock — "AI 에이전트 안에 우리 매장을 여는 미들웨어"

브랜드 카탈로그를 AI가 이해하는 형태로 변환하고, 모든 AI 플랫폼에 한 번에 배포합니다

🧩 Lemrock이 하는 일, 한마디로

"브랜드의 상품 카탈로그를 ChatGPT, Claude, Perplexity 같은 AI 에이전트 안에서 검색·추천·구매 가능하게 만들어주는 플랫폼"

비유하면, 네이버 스마트스토어나 쿠팡에 입점하듯이, AI 에이전트에 입점하는 것을 도와주는 서비스입니다. 다만 입점 대상이 웹사이트가 아니라 ChatGPT의 대화창입니다.

STEP 1
카탈로그 연결

→
STEP 2
AI-Ready 변환

→
STEP 3

ChatGPT, Claude, Perplexity 등 모든 AI에 동시 배포
→
STEP 4
AI 대화 내 노출
사용자 질문 맥락에 맞춰 개인화된 상품 추천·미니 스토어
→
STEP 5
성과 추적
노출·클릭·전환 지표를 브랜드 대시보드에서 실시간 확인

각 단계를 자세히 이해하기

단계구체적으로 무엇을 하는가왜 필요한가
1. 카탈로그 연결 브랜드의 기존 상품 DB — Shopify, Magento, 자체 PIM, ERP 등 — 에서 API 또는 피드로 상품 정보(제목, 가격, 재고, 이미지, 설명 등)를 수집합니다. 실시간 가격·재고 변동도 자동 동기화됩니다. 브랜드가 이미 갖고 있는 데이터를 그대로 활용하므로, 별도 데이터 작업 없이 바로 시작할 수 있습니다.
2. AI-Ready 변환 Lemrock의 독자 AI 모델이 상품 데이터를 분석해 — ① 대화형 속성 생성 (best_for, FAQ 등) ② 마케팅 문구 정제 ③ 각 플랫폼별 규격(ACP/UCP 등)에 맞춘 구조화. 2026년 5월 현재 UCP의 Cart·Catalog·Identity Linking과 ACP의 Stripe SPT 규격을 모두 자동 생성하여, 브랜드가 프로토콜별 개발 없이 듀얼 프로토콜 대응이 가능합니다. 대화 맥락(선호도, 이전 질문, 제약조건)을 분석하여 초개인화 추천을 지원합니다. "LG 그램 17 특가!!!" → "LG 그램 17, 17인치, 1.35kg, 출장용 노트북"으로 변환. AI가 정확하게 이해할 수 있는 형태가 됩니다.
3. 멀티 배포 변환된 데이터를 ChatGPT(ACP), Google AI Mode(UCP), Claude, Perplexity, Gemini 등 모든 주요 AI 플랫폼에 동시 배포. Lemrock은 LLM-agnostic(특정 AI에 종속되지 않음) 벤더 중립 미들웨어를 표방하며, UCP+ACP 프로토콜 스택 시대에 "한 번 연동으로 듀얼 프로토콜 대응"이 핵심 가치입니다. 브랜드가 각 플랫폼에 개별 연동하면 수개월과 수억원이 듭니다. Lemrock은 이걸 "한 번 연동"으로 해결합니다.
4. 대화 내 노출 사용자가 AI에게 "여행용 가벼운 노트북 추천해줘"라고 물으면, Lemrock이 맥락을 분석하여 해당 브랜드의 적합 상품을 이미지·가격·재고 포함 "미니 스토어" 형태로 대화 안에 노출합니다. 단순 텍스트 추천이 아니라, 이미지와 가격이 포함된 쇼핑 경험을 AI 대화 안에서 제공합니다.
5. 성과 추적 어떤 AI 플랫폼에서, 어떤 질문 맥락에서, 몇 번 노출되고, 몇 번 클릭되고, 몇 건 전환되었는지 브랜드 전용 대시보드에서 실시간 확인. 마치 Google Ads 대시보드처럼. 브랜드가 AI 채널의 ROI를 측정하고, 데이터 기반으로 상품 정보를 최적화할 수 있습니다.

💰 수익 모델

Lemrock은 성과 기반 과금을 채택합니다. 브랜드와 협상에 따라 — ① 판매 수수료(거래 성사 시) ② 노출 기반(임프레션/발견 횟수) ③ 전환 기반(구매 퍼널 깊이에 따른 성과 지표) 중 조합으로 과금합니다. 즉 "효과가 있어야 돈을 내는" 구조입니다.

🛒 Rye가 하는 일, 한마디로

"어떤 쇼핑몰의 상품이든, 상품 URL만 주면 AI가 알아서 장바구니에 담고 결제까지 완료해주는 API"

비유하면, "만능 대리 구매 로봇"입니다. 당신이 원하는 상품의 웹페이지 주소만 알려주면, 로봇이 그 사이트에 가서 사람처럼 상품을 장바구니에 담고, 배송지를 입력하고, 결제를 완료한 뒤, 주문 확인 번호를 가져다줍니다.

❓ 왜 Rye가 필요한가? — "프로토콜의 한계"

ACP(OpenAI)는 판매자가 직접 피드를 만들어 보내야 합니다. UCP(Google)는 판매자가 .well-known/ucp를 구현해야 합니다. 문제는 전 세계 2억 개 이상의 온라인 쇼핑몰 중 99%는 이런 프로토콜을 영원히 구현하지 않을 것이라는 점입니다. 동네 빵집 온라인 스토어, 소규모 수공예 쇼핑몰, 중소 제조업체 홈페이지... 이런 곳은 ACP 피드를 만들 엔지니어가 없습니다.

Rye의 발상: "이미 모든 머천트가 채택한 프로토콜이 있다 — 바로 웹(WWW) 그 자체". 모든 쇼핑몰에는 장바구니와 체크아웃 페이지가 있습니다. Rye는 이 기존 웹 체크아웃을 AI가 자동으로 수행하게 합니다.

Rye 작동 방식 — 3-Layer 아키텍처

Layer 1: API 레이어

개발자가 호출하는 인터페이스

2단계 계약: ① Intent 생성 (상품 URL + 결제 토큰 전달) → ② Offer 수신 (실시간 가격·배송비·세금 확인) → ③ Confirm (주문 확정)

↕
Layer 2: AI 브라우저 에이전트

사람처럼 웹을 탐색하는 AI

상품 페이지 방문 → 장바구니 담기 → 배송지 입력 → 쿠폰/할인 확인 → 결제 정보 입력 → 주문 완료. 실제 브라우저를 조작하여 체크아웃 플로우를 자동 수행합니다.

↕
Layer 3: 최적화 & 사기방지

속도와 안정성을 보장하는 인프라

① 결정론적 워크플로우: 첫 성공 시 AI 행동을 기록 → 다음부터 동일 사이트는 고속 재생 (5초 이내)
② 사기방지 프록시: 거주지 IP 선택, 지리적 근접성, 사람 유사 행동 패턴으로 주문 취소 방지
③ 자가 치유: 사이트 레이아웃이 변경되면 AI가 자동 적응

Rye의 두 가지 API 제품

📣 Rye의 선언: "The Protocol" (2026.03.26)

Rye는 ACP, UCP, TAP 등 프로토콜 난립 상황에 대해 "이미 모든 머천트가 채택한 프로토콜이 있다 — WWW 그 자체"라고 선언했습니다. 200만 개 이상의 엔드포인트, 30년 이상의 실전 검증, 모든 브라우저·디바이스·국가가 지원하는 웹 자체가 곧 프로토콜이라는 관점입니다.

실질적으로 이것은 "프로토콜을 기다릴 필요 없이, 지금 당장 어떤 쇼핑몰에서든 AI가 구매할 수 있다"는 의미입니다. 실제로 OpenAI가 ChatGPT 네이티브 체크아웃을 12개 머천트 단계에서 축소한 사건이 Rye의 관점을 뒷받침합니다.

🔐 신규: Skyfire KYA 통합 — 에이전트 신원 인증 레이어 (2026.Q1)

Rye는 Skyfire의 KYA(Know Your Agent) 프로토콜을 Universal Checkout에 네이티브 통합했습니다. KYA는 AI 에이전트의 신원을 검증하여, 머천트의 봇 방어 인프라(Akamai, F5, Datadome, Imperva 등)가 정당한 에이전트 트래픽을 허용하도록 합니다.

개발자는 Rye API 하나로 결제 + 에이전트 신원 인증을 동시에 처리할 수 있게 되었습니다. 이는 UCP+ACP 프로토콜 스택 시대에 "프로토콜 미구현 쇼핑몰"에서의 결제를 더욱 안정적으로 만듭니다.

4 Lemrock vs Rye — 직접 비교

같은 "미들웨어"지만 푸는 문제가 완전히 다릅니다

🍽️ 비유: 레스토랑 비즈니스로 이해하기

📋

Lemrock = "메뉴판 제작 & 배달앱 등록 대행"

레스토랑의 음식 사진을 찍고, 맛있어 보이게 편집하고, 배달의민족·쿠팡이츠·요기요에 동시 등록해주고, 어느 앱에서 얼마나 주문이 들어오는지 알려줍니다.

vs
🏍️

Rye = "만능 배달 라이더"

어떤 레스토랑이든(배달앱에 등록 안 되어 있어도!) 직접 가서 음식을 주문하고, 결제하고, 고객에게 배달해줍니다. 레스토랑이 배달 시스템을 따로 갖출 필요가 없습니다.

5 같이 쓰면? — 에이전틱 커머스 풀스택

Lemrock + Rye + Syndigo/Akeneo가 조합되면 어떤 그림이 되는가

💡 핵심 인사이트: "이 회사들은 경쟁이 아니라 보완 관계"

에이전틱 커머스의 가치 사슬은 데이터 정비 → 노출·발견 → 결제·이행의 3단계로 구성됩니다. 각 회사는 서로 다른 단계를 담당합니다. UCP+ACP 듀얼 프로토콜 시대에 Lemrock은 프로토콜 대응 레이어를, Rye+Skyfire는 프로토콜 미구현 쇼핑몰의 결제+신원인증 레이어를 각각 담당하여, 프로토콜 채택 여부와 무관하게 전체 커머스 커버리지를 달성합니다.

① 데이터 정비

② 노출 & 발견

6 한국 시사점 & 기회

Lemrock과 Rye 모델에서 한국 시장이 배울 것

🇰🇷 Lemrock 모델 → 한국에서의 기회

"한국형 AI 커머스 입점 미들웨어"

네이버 스마트스토어, 쿠팡, 카카오 쇼핑 등 국내 플랫폼의 상품 데이터를 — ChatGPT(ACP), Google AI Mode(UCP), 네이버 쇼핑 AI 에이전트, 카카오 PlayMCP 등에 동시 배포하는 미들웨어. UCP/ACP 듀얼 프로토콜 자동 대응 + 한국어 NLP 특화 + 국내 PG(이니시스, 토스페이먼츠) 연동이 차별화 요소. 초기 데이터에서 듀얼 구현 가맹점이 40% 더 많은 에이전트 트래픽을 확보하는 만큼, 한국 브랜드의 프로토콜 대응을 대행하는 미들웨어의 가치가 큽니다.

→ 이전 리포트의 "영역 2: 에이전틱 커머스 미들웨어" 사업 모델에 해당

🇰🇷 Rye 모델 → 한국에서의 기회

"한국 쇼핑몰 만능 결제 API"

한국의 수십만 개 스마트스토어, 카페24 쇼핑몰, 자체 쇼핑몰은 ACP/UCP를 절대 구현하지 않을 것입니다. OpenAI조차 ChatGPT 체크아웃을 12개 머천트에서 축소한 현실이 이를 증명합니다. 이런 쇼핑몰에서도 AI가 주문을 완료할 수 있는 브라우저 자동화 체크아웃 API + KYA 방식의 에이전트 신원 인증이 필요합니다. 한국 특유의 본인인증, PG 결제 창, 주소 검색 등을 자동화하는 것이 핵심 장벽이자 해자.

→ 한국 PG 격리, 본인인증, iamport 연동 등이 글로벌 플레이어의 한국 진입 장벽

🔗 에이전틱 커머스 가치 사슬에서의 한국 빈자리

Syndigo/Akeneo 포지션 (데이터 정비) → 한국 부재 → 한국형 AI-Ready 상품 피드 인프라 필요 (리포트 #8 참조)
Lemrock 포지션 (노출/발견) → 한국 부재 → 한국형 AI 커머스 입점 미들웨어 필요
Rye 포지션 (결제/이행) → 한국 부재 → 한국형 만능 결제 API 필요

세 영역 모두 한국에서는 아직 본격적인 플레이어가 없습니다. 토스페이먼츠 MCP Server가 결제 영역에서 가장 가까운 시도이고, 와들(젠투)이 노출/발견 영역에서 가능성을 보이고 있지만, UCP+ACP 듀얼 프로토콜 대응과 KYA 에이전트 신원 인증까지 포괄하는 전체 가치 사슬 통합 미들웨어는 아직 없습니다.

7 용어 정리

미들웨어 (Middleware)
두 시스템(여기서는 브랜드 ↔ AI 에이전트) 사이를 연결하는 중간 소프트웨어. 양쪽의 차이를 흡수하여 호환성을 제공합니다.
LLM-Agnostic
특정 LLM(GPT, Claude, Gemini 등)에 종속되지 않는 구조. Lemrock은 어떤 AI 플랫폼에도 연결 가능합니다.
브라우저 자동화 (Browser Automation)
AI가 실제 웹 브라우저를 조작하여 사람처럼 웹사이트를 탐색하는 기술. Rye의 핵심 기술입니다.
결정론적 워크플로우 (Deterministic Workflow)
AI 브라우저가 한 번 성공한 체크아웃 패턴을 기록해두고, 다음부터 동일 사이트에서는 AI 없이 고속 재생하는 방식.
Landed Cost (총 지불 비용)
상품 가격 + 배송비 + 세금 + 관세를 모두 합한 실제 소비자 지불 금액. Rye가 Offer에서 제공하는 핵심 정보.
토큰화 결제 (Tokenized Payment)
실제 카드번호 대신 일회용 토큰을 사용하여 결제하는 방식. AI 에이전트에게 카드 정보를 직접 노출하지 않아 보안성 확보.
카탈로그 신디케이션 (Catalog Syndication)
하나의 상품 카탈로그를 여러 플랫폼에 동시 배포하는 것. Lemrock과 Shopify Catalog이 수행하는 핵심 기능.
프로토콜 파편화 (Protocol Fragmentation)
ACP, UCP, TAP, Agent Pay 등 여러 표준이 난립하여 판매자 부담이 가중되는 현상. 미들웨어의 존재 이유.
KYA (Know Your Agent)
Skyfire가 개발한 AI 에이전트 신원 인증 프로토콜. 머천트의 봇 방어 인프라(Akamai, F5, Datadome 등)가 정당한 에이전트를 식별하도록 KYA 토큰을 발급합니다. Rye에 네이티브 통합.
듀얼 프로토콜 (Dual Protocol)
UCP와 ACP를 동시에 구현하는 전략. 2026년 초기 데이터에서 듀얼 구현 가맹점이 단일 프로토콜 대비 40% 더 많은 에이전트 트래픽을 확보.
